# Telegram-управляемый диспенсер на Raspberry Pi Pico W

## Обзор проекта

Данный проект представляет собой систему на MicroPython для Raspberry Pi Pico W.  
Устройство подключается к Wi-Fi, запускает Telegram-бота и управляет аппаратными компонентами: встроенным светодиодом, кнопкой и шаговым двигателем.

Основная цель проекта — показать взаимодействие микроконтроллера с Telegram и физическим оборудованием в режиме реального времени.

---

# Основные возможности

- Подключение к Wi-Fi
- Интеграция с Telegram Bot API
- Обработка команд через Telegram
- Поддержка inline-кнопок
- Получение уникального ID устройства
- Обработка состояния кнопки
- Управление светодиодом
- Управление шаговым двигателем через ULN2003
- Управление механизмом диспенсера

# Подробное описание файлов проекта

## main.py

`main.py` является главным файлом системы и основной точкой входа проекта.  
После запуска устройства именно этот файл начинает выполнение всей программы.

Файл отвечает за:

- импорт всех необходимых модулей проекта
- подключение Raspberry Pi Pico W к Wi-Fi сети
- инициализацию Telegram-бота
- создание объектов GPIO
- запуск бесконечного цикла работы устройства
- проверку сообщений Telegram
- обработку состояния кнопки
- управление встроенным LED-индикатором

### Основная логика работы

После включения устройство автоматически подключается к сети Wi-Fi с использованием данных из `secrets.py`.

Затем система:

1. запускает Telegram-бота
2. создаёт объекты кнопки и LED
3. начинает бесконечный цикл работы
4. периодически проверяет новые сообщения Telegram
5. реагирует на команды пользователя
6. управляет состоянием оборудования

### Используемые компоненты

| Компонент | Назначение |
|---|---|
| Wi-Fi | Подключение к интернету |
| Telegram Bot | Удалённое управление |
| Button | Обработка пользовательского ввода |
| LED | Индикация состояния |

### Особенности

- используется неблокирующая логика
- проверка Telegram выполняется по таймеру
- код адаптирован под MicroPython
- управление GPIO происходит в реальном времени

---

## bot.py

Файл `bot.py` содержит всю логику Telegram-бота.

Именно здесь:

- создаётся объект бота
- регистрируются команды
- обрабатываются сообщения
- создаются inline-кнопки
- формируются ответы пользователю

### Основные функции файла

### Команда `/start`

Отправляет пользователю приветственное сообщение и информацию о системе.

### Команда `/help`

Выводит список доступных команд и описание возможностей бота.

### Команда `/btn`

Создаёт inline-кнопки Telegram для взаимодействия с устройством.

### Команда `/unique_id`

Возвращает уникальный идентификатор Raspberry Pi Pico W.

---

### Callback-функции

Файл также содержит callback-обработчики для inline-кнопок.

Например:

- получение информации об устройстве
- получение ID устройства
- обработка действий пользователя

---

### Дополнительные возможности

Если пользователь отправляет обычный текст, бот автоматически отвечает тем же сообщением.

Это позволяет проверить:

- работу Telegram API
- подключение к интернету
- стабильность соединения

## telegram.py

`telegram.py` является собственной реализацией Telegram Bot API для MicroPython.

Данный модуль играет ключевую роль проекта, так как стандартные библиотеки Telegram не предназначены для работы на микроконтроллерах с ограниченными ресурсами.

### Основные задачи файла

- взаимодействие с Telegram API
- отправка HTTP-запросов
- получение обновлений
- обработка команд
- обработка callback-кнопок
- работа с сообщениями

---

### Реализованные функции

| Функция | Назначение |
|---|---|
| send_message() | Отправка сообщений |
| reply() | Ответ на сообщение |
| get_updates() | Получение новых сообщений |
| delete_message() | Удаление сообщений |
| command() | Регистрация Telegram-команд |
| callback() | Обработка inline-кнопок |

---

### Особенности реализации

- адаптация под MicroPython
- минимальное использование памяти
- работа через HTTPS-запросы
- поддержка callback-кнопок
- поддержка reply и inline клавиатур

---

### Значение для проекта

Без данного файла Raspberry Pi Pico W не смог бы взаимодействовать с Telegram.

Файл фактически реализует сетевой интерфейс между Telegram и микроконтроллером.

---

## helper.py

Файл `helper.py` содержит вспомогательные функции для работы со временем и интервалами.

Он используется для организации неблокирующей логики работы системы.

---

### Функция is_interval_elapsed()

Проверяет, прошло ли указанное время с последнего события.

Используется для:

- периодической проверки Telegram
- выполнения циклических действий
- обновления состояний системы

---

### Функция true_for_interval()

Удерживает логическое значение `True` в течение заданного времени после появления сигнала.

Используется для:

- обработки кнопок
- фильтрации дребезга контактов
- временного удержания состояния

---

### Преимущества подхода

Использование таймеров вместо `sleep()` позволяет:

- не блокировать выполнение программы
- одновременно выполнять несколько задач
- повысить отзывчивость системы

## motor.py

Файл `motor.py` реализует библиотеку управления шаговым двигателем через драйвер ULN2003.

---

### Назначение

Файл отвечает за:

- управление шагами двигателя
- управление направлением вращения
- позиционирование двигателя
- хранение текущего положения

---

### Реализованные классы

| Класс | Описание |
|---|---|
| Motor | Базовый класс двигателя |
| FullStepMotor | Полношаговый режим |
| HalfStepMotor | Полушаговый режим |

---

### Full Step Mode

В полном шаге двигатель делает более крупные шаги.

Преимущества:

- выше скорость
- проще управление

Недостатки:

- ниже точность
- больше вибраций

---

### Half Step Mode

В полушаговом режиме двигатель работает плавнее.

Преимущества:

- более высокая точность
- более плавное движение

Недостатки:

- ниже скорость
- сложнее управление

---

### Основные методы

#### step()

Выполняет заданное количество шагов.

#### step_until()

Перемещает двигатель в указанную позицию.

#### step_until_angle()

Поворачивает двигатель на заданный угол.

#### zero()

Сбрасывает текущую позицию двигателя.

## Workflow взаимодействия с Telegram

1. Пользователь отправляет команду через Telegram.

2. Telegram сохраняет update на сервере.

3. Raspberry Pi Pico W периодически опрашивает Telegram API.

4. Полученная команда анализируется системой.

5. Вызывается соответствующая функция-обработчик.

6. Выполняется управление оборудованием.

7. Пользователь получает ответ от бота.
